Ellegant CSS Framework

We've been using style frameworks in the past decades which are supposed to focus on CSS. But the truth is there are a lot of Javascript code being used that we could replace with some clever CSS selectors to replicate it's function. Ellegant's goal is to allow CSS to be used for everything it is capable of.

General Components

General Components react to user interaction so you can use them for a variety of different purposes.

  1. The javascript part is only needed if you want ARIA roles;
  2. The style-* classes are used only for visual demos, you don't actually need them;
  3. The ID should be unique to avoid conflicts;
  4. The elle-* classes and for attributes are needed.

Toggler

A Toggler is a component that allows you to show and hide elements based on user interaction using checkboxes and labels.

  • Another one for good measure

    <section class="elle-toggler">
        <label class="elle-label style-elle-btn" for="myToggler" role="switch" aria-checked="false">Toggler</label>
        <input type="checkbox" class="elle-checkbox" id="myToggler">
        <ul class="elle-target">
        <li><label for="myToggler" role="switch" aria-checked="true">Click <strong>here</strong> to also toggle it from inside</label></li>
        <li>An item</li>
        <li>Another one for good measure</li>
        </ul>
    </section>
